In early April 2026, the Council executed a full migration from Playwright to **Agent Browser**. The goal wasn't just to replace a library — it was to eliminate "context bloat," the primary bottleneck in autonomous browser interaction.

Traditional automation tools (Playwright, Puppeteer) provide agents with the full DOM. For a complex site like LinkedIn, this can exceed 50,000 tokens. Agent Browser replaces this with a **snapshot-based reference system**, reducing that same page to ~20 lines of interactive elements.

## Why Agent Browser?

<CardGrid>
  <Card title="Context Efficiency" icon="rocket">
    Snapshots are up to 99% smaller than raw HTML, saving thousands of tokens per interaction.
  </Card>
  <Card title="Reference IDs" icon="list-format">
    Elements are assigned short IDs (like `@e1`, `@e2`), making interaction simple: `agent-browser click @e1`.
  </Card>
  <Card title="Native Iframes" icon="laptop">
    Iframe content is automatically inlined into the main tree, eliminating the need for complex frame switching.
  </Card>
  <Card title="Session Persistence" icon="lock">
    Supports named sessions and encrypted auth states, allowing agents to stay logged into LinkedIn or other services.
  </Card>
</CardGrid>

## Core Workflow

Every browser task in Sanctum now follows this pattern:

<Steps>
1.  **Navigate**: `agent-browser open <url>`
2.  **Snapshot**: `agent-browser snapshot -i` (returns element refs like `@e1`, `@e2`)
3.  **Interact**: Use refs to click, fill, or select: `agent-browser fill @e2 "Search query"`
4.  **Verify**: Re-snapshot or take a screenshot to confirm the action succeeded.
</Steps>

## Sector Integrations

The migration impacted three critical monitoring sectors:

- **LinkedIn Affinity**: Now uses `agent-browser` with cookie-based session state to monitor selector stability and extract profile names.
- **OBLITERATUS**: Verifies Gradio UI hydration by checking the character density of the snapshot.
- **Holocron UI**: Uses iframe inlining to verify the Command Center (port 1111) is correctly rendering inside the main UI (port 3333).

## E2E Validation

The integration is verified by `tests/test-agent-browser-e2e.sh`, which validates:
- CLI installation and version consistency.
- Navigation and snapshot density (~18 lines for Google).
- Screenshot capture and storage.
- Named session isolation.

<Aside type="tip">
When elements are tricky or dynamic, use `agent-browser screenshot --annotate` to see a numbered overlay of every interactive element on the page.
</Aside>
