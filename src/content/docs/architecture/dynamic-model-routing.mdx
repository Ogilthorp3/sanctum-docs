Hardcoding paths in LaunchAgents is a recipe for a 3 AM troubleshooting session where you realize you're running a model from last March because you forgot to update a plist. We've moved the brain's coordinates to a single source of truth.

## The Single Source of Truth

All model and adapter paths are now centralized in `~/.sanctum/instance.yaml`. This is the core configuration for the house. If it isn't in here, the system doesn't know it exists.

```yaml
# Actual configuration from instance.yaml
services:
  mlx_server:
    enabled: true
    port: 1337
    model_path: ~/Projects/mlx-finetune/models/Qwen3.5-27B-Claude-Opus-Distilled-v2-4bit
    adapter_path: ~/Projects/mlx-finetune/adapters-carmack-overnight
    venv: ~/Projects/mlx-finetune/.venv
```

<Aside type="note">
The port is 1337 because, let's face it, running a 27B model on a Mac Mini is exactly that.
</Aside>

## The Dynamic Wrapper

The MLX server no longer receives its orders from a static plist. Instead, it uses a dynamic wrapper script that parses the configuration at runtime.

### How It Works

<Steps>
1.  **Parse YAML:** The wrapper uses `yq` to extract the current active model, adapter, and port from `instance.yaml`.
2.  **Environment Isolation:** It sets `PYTHONPATH` and `VIRTUAL_ENV` to point to the specific virtual environment required for the model.
3.  **The Shim:** It prepends a shim directory to the `PATH` to force the `sanctum-server` binary to use the venv's Python interpreter instead of whatever Homebrew decided was the default this week.
4.  **Offline Enforcement:** It sets `HF_HUB_OFFLINE=1`. We don't need the model trying to check its own birth certificate on HuggingFace every time it starts.
</Steps>

## Proxy Integration

The `sanctum-proxy` (port 4040) is the air traffic controller. It routes requests to the MLX server on its dynamic port.

```yaml
# sanctum-proxy/config.yaml
- name: council-heartbeat
  provider: local
  api_model: /Users/neo/Projects/mlx-finetune/models/Qwen3.5-27B-Claude-Opus-Distilled-v2-4bit
  api_base: http://10.10.10.1:1337
```

<Aside type="caution">
If the proxy and the server disagree on the port, the proxy will fallback to LM Studio, which will proceed to eat 22GB of your RAM while you're not looking.
</Aside>

## Triage & Recovery

**Qui-Gon Triage** (`qui_gon_triage.py`) monitors the health of these services. If it finds a service is down, it uses `launchctl kickstart` to revive it. Because of the dynamic wrapper, the revived service always wakes up with the latest settings from `instance.yaml`.
