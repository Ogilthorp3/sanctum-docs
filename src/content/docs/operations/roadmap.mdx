## Next — Queued

<CardGrid>
<Card title="Frigate NVR — Context-Aware Alerts" icon="seti:video">
Ring doorbells detect motion but cannot tell a raccoon from an intruder. Frigate provides AI object detection (person, animal, vehicle, package) via camera RTSP feeds. Force Flow receives the object class and sets severity accordingly: person = critical, animal = info, vehicle = warn. The raccoon never reaches your phone.

**Architecture:** Ring RTSP → Frigate (Docker + Google Coral TPU) → HA automation → Force Flow (4077)

**Effort:** Full session. Frigate setup, RTSP config, HA integration, severity mapping.
</Card>

<Card title="Force Flow — Screen Time & Parental Controls" icon="seti:clock">
The flagship feature of Force Flow beyond notifications: regulate screen access across the house to help parents build healthy habits for their children. Force Flow already controls what the house says and when — the natural extension is controlling what the screens show and for how long.

**Capabilities:**
- Per-child screen time budgets (weekday vs weekend, school vs vacation)
- Content category restrictions (gaming, streaming, social media, educational)
- Automatic enforcement via network-level controls (Firewalla API + HA automations)
- Gentle Sonos announcements when time is running out ("Albert, 10 minutes remaining")
- Parental override via Force Flow API or iPhone notification actions
- Weekly reports through Force Flow history (total screen time, categories, trends)

**Architecture:** Force Flow (4077) → Firewalla API (device blocking) + HA automations (Sonos announcements) + iPhone push (parental controls). Per-child profiles in instance.yaml. The same quiet hours engine that stops raccoon alerts at night stops YouTube at bedtime.

**Effort:** Multi-session. Device-to-child mapping, content categorization via DNS/Firewalla, enforcement rules, parental UI, Sonos integration.
</Card>

<Card title="Test Suite — Instance-YAML Aware" icon="seti:config">
Half the pre-existing test failures are tests expecting services that are intentionally disabled. The council-router test suite should read `instance.yaml` and skip tests for disabled services. One architectural change, five fewer false failures, and daily test reports that are actually trustworthy.

**Effort:** Medium. Modify test-framework.sh to parse instance.yaml, update each suite.
</Card>
</CardGrid>

## Soon — This Quarter

<CardGrid>
<Card title="Bare QEMU — Apple Dev ID Signing" icon="seti:lock">
When Bert's Apple Developer account is approved, sign the QEMU binary with the `com.apple.vm.networking` entitlement. This allows QEMU to use `vmnet-host` directly instead of going through `socket_vmnet`. Simpler architecture, one fewer root daemon, one fewer socket in the chain.

**Blocked by:** Apple Developer Program enrollment.
</Card>

<Card title="Rust MLX Server" icon="seti:rust">
Crash-proof Rust replacement for the Python `mlx_lm.server` on port 1337. The Python server occasionally hangs under Metal command buffer pressure. A Rust wrapper with health monitoring, auto-restart, and proper signal handling would eliminate the most common Council MLX failure mode.

**Effort:** Full session. Rust + MLX FFI, health endpoint, LaunchAgent.
</Card>

<Card title="Jump Desktop Reliability" icon="seti:image">
Jump Desktop failed to connect twice during the Raccoon Theorem session. Investigate root cause — may need Tailscale SSH as fallback, or a Cloudflare Tunnel to a VNC endpoint. Remote GUI access must be reliable for the bootstrap fallback (Keychain unlock, UTM debugging).

**Effort:** Investigation + config. Not a code project.
</Card>
</CardGrid>

## Later — Backlog

<CardGrid>
<Card title="Council Autoresearch" icon="seti:python">
Karpathy-style autonomous LoRA fine-tuning for the Sanctum council agents. The system identifies knowledge gaps, generates training data, runs MLX fine-tuning, and evaluates results — all without human intervention. The council teaches itself.

**Effort:** Multi-session project. Data pipeline, training harness, evaluation framework.
</Card>

<Card title="27B LoRA Abliteration" icon="seti:crystal">
Abliteration strategy for Qwen3.5-27B-4bit — removing refusal behavior while preserving capability. Requires stable Metal command buffer handling (currently crashes under sustained fine-tuning load).

**Blocked by:** Metal stability on M4 Pro under LoRA training.
</Card>

<Card title="Signal Delivery Hardening" icon="seti:lock">
Signal CLI containers (ports 18081, 18082) have intermittent connectivity issues. The Docker port forwarding through the bridge is unreliable. May need to move Signal to a native Mac process or a dedicated socat proxy.

**Effort:** Medium. Diagnosis + architecture decision.
</Card>

<Card title="Dashboard Alias (haus)" icon="seti:html">
The `http://haus/` DNS alias returns 502. The Dench Proxy LaunchDaemon (port 80) was disabled. Either re-enable it or set up a Cloudflare Tunnel route for local dashboard access.

**Effort:** Small. Config change.
</Card>
</CardGrid>
